# alu-system_engineering-devops

ALU System Engineering & DevOps track projects: setting up, securing and
load-balancing a small web stack (SSH access, nginx web servers, HAProxy,
ufw firewall) and debugging a series of intentionally broken web stacks.

## Projects

- [ssh](ssh/README.md) — key-based access to the assigned servers
- [web_server](web_server/README.md) — nginx web server setup via Bash
- [load_balancer](load_balancer/README.md) — redundant web servers behind HAProxy
- [firewall](firewall/README.md) — locking down web-01 with ufw
- [web_stack_debugging_0](web_stack_debugging_0/README.md) — Apache not running
- [web_stack_debugging_1](web_stack_debugging_1/README.md) — nginx on the wrong port
- [web_stack_debugging_2](web_stack_debugging_2/README.md) — running services as non-root
- [web_stack_debugging_3](web_stack_debugging_3/README.md) — a WordPress 500 error, fixed with Puppet
- [web_stack_debugging_4](web_stack_debugging_4/README.md) — nginx failing under load
